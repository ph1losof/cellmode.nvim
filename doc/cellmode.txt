*cellmode.txt*  cellmode.nvim - visual table editing for raw CSV/TSV

INTRODUCTION                                          *cellmode*

cellmode is a Neovim plugin that gives CSV and TSV files a column-aligned
visual presentation while keeping the buffer contents identical to the bytes
on disk. Column separators and alignment padding are pure visual decoration
rendered with extmarks (inline virtual text plus conceal); they are never
written to the buffer.

REQUIREMENTS                                          *cellmode-requirements*

- Neovim 0.10 or newer (inline `virt_text` is required)
- No external dependencies

ARCHITECTURE                                          *cellmode-architecture*

- The buffer always contains raw CSV/TSV bytes. `:w` is a normal Vim write;
  there is no BufWriteCmd, no adapter subprocess, no model-to-buffer
  projection.
- |cellmode.codec.csv_parser| parses buffer lines into records with byte
  spans per field. RFC 4180 quoting and embedded newlines are honored;
  multi-line cells span multiple buffer lines.
- |cellmode.view.cell_layout| caches the parsed records and per-column
  display widths per buffer. On each edit the affected range is reparsed and
  widths are recomputed; if widths change the overlay is fully redrawn,
  otherwise only the touched records are redrawn.
- |cellmode.view.overlay| places extmarks via a single namespace
  (`cellmode_overlay`):
  * a full column grid on every physical buffer row of a record,
  * inline padding + pipe at every delimiter position, plus empty padded
    cells for columns that wrap onto other lines so rows stay aligned,
  * `conceal=""` over delimiters, surrounding quotes, and one byte of each
    escaped `""` pair so it shows a single `"`.
  Window-local options `conceallevel=2` and `concealcursor=nc` are set on
  attach so editing a line reveals the raw bytes while normal-mode viewing
  shows the pretty grid.
- |cellmode.runtime.auto_quote| watches `TextChangedI`. When the active cell
  is unquoted and the user types the format's delimiter, the cell is wrapped
  in quotes so cell boundaries stay stable.

COMMAND                                               *:Cellmode*

  :Cellmode open <path> <csv|tsv>
  :Cellmode op set-cell <row> <col> <value>
  :Cellmode op insert-row <row> <value1,value2,...>
  :Cellmode op delete-row <row>
  :Cellmode toggle
  :Cellmode save
  :Cellmode status

`<row>` and `<col>` are 1-based indexes into the parsed records / fields.
`set-cell` rewrites the buffer text covering the cell; values are
auto-quoted when they contain the delimiter, a quote, or a newline.

SETUP                                                 *cellmode-setup*

>
  require("cellmode").setup({
    command = "Cellmode",
    marks = {
      pipe = "│",
    },
  })
<

The plugin attaches automatically on `BufReadPost` for buffers whose
filetype or extension is `csv` or `tsv`. No explicit `:Cellmode open` call
is required for normal use.

HIGHLIGHTS                                            *cellmode-highlights*

`CellmodePipe`         column separator glyph
`CellmodePadding`      alignment padding
`CellmodeHbar`         reserved (row underline)
`CellmodeSpecialChar`  reserved (special-character glyph)

All groups are defined with `default = true`. Override via
`vim.api.nvim_set_hl(0, "CellmodePipe", { ... })`.

TESTING                                               *cellmode-testing*

>
  ./scripts/test/run-render.sh
<

Headless integration suite covering session attach, raw-CSV buffer
contents, layout, extmark placement, cell ops, overlay toggle, and save
round-trip.
